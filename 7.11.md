# 版本控制系统

可回档
> 可以回到任意历史版本，不怕改错。

凡事留痕
> 每次修改都有记录：谁改的、什么时候改的、改了哪里。

## SVN

代码提交到中央仓库
> SVN 是集中式版本控制，所有代码存在中央服务器，提交时必须联网。

## Git

分布式，离线可提交
> 每个开发者本地都有完整仓库副本，不联网也能 commit，等有网时再 push。


### 初始化仓库

```
git init
```
> 在当前目录创建一个空的 Git 仓库（生成 .git 目录）。

### 添加到暂存区

```
git add .
```
> 将当前目录下所有修改加入暂存区（staging area），准备提交。

### 提交暂存区代码

```
git commit -m ""
```
> 将暂存区的内容提交到本地仓库，`-m` 后写提交说明。

### 添加远程仓库

```
git remote add $仓库别名 $远程仓库URL
```
> 给本地仓库关联一个远程地址，通常别名用 `origin`。

### 推送到远程仓库

```
git push $仓库别名 $分支
```
> 将本地提交推送到远程仓库，让其他人能看到。

### 克隆远程仓库

```
git clone $远程仓库URL
```
> 把远程仓库完整下载到本地，包含所有历史记录。

### 拉取仓库，需要合并冲突

```
git pull
```
> 从远程拉取最新代码并自动合并到当前分支。如果有冲突需要手动解决。

### 创建分支

```
git branch $分支
```
> 创建新分支，用于并行开发，互不干扰。

### 选择分支

```
git switch $分支
```
> 切换到指定分支，工作区内容会变成该分支的代码。

### 合并分支

```
git merge $分支
```
> 将指定分支的修改合并到当前分支。

## 简介

* 跨平台
> 一套代码可运行在 Windows、Linux、macOS 等多个平台。

* 多线程
> Qt 提供线程类（QThread），支持并发编程。

* 组件化
> Qt 由多个模块组成（如 QtCore、QtWidgets），按需引入。

* 信号槽
> Qt 特有的对象间通信机制，信号发出时自动调用连接的槽函数，替代传统回调。
